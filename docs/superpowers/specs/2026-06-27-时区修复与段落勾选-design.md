# 时区修复与段落临时勾选 设计文档

> 日期：2026-06-27
> 范围：统计页时区错位修复 + 近 7 天每日 Token 趋势 + 生成反馈时段落临时勾选

## 1. 目标

修复两个用户报告的问题，并补齐相关体验：

1. **时间不对**：统计页按日期聚合 Token 消耗时，中国时区（UTC+8）下凌晨 0–8 点生成的反馈被错误归到前一天；且 byDay 是静态列表，无"每日"概念。
2. **段落不能自选**：规范档的段落（segments）在生成反馈时被强制全部塞进 prompt，老师无法本次临时排除某个段落。

## 2. 现状与问题

### 2.1 时间问题

[src/hooks/useStats.ts#L22](file:///workspace/src/hooks/useStats.ts#L22) 当前实现：

```ts
const day = new Date(u.timestamp).toISOString().slice(0, 10);
```

`toISOString()` 返回 UTC 时间。中国时区 UTC+8，凌晨 00:00–08:00（北京时间）对应的 UTC 是前一天 16:00–24:00，被切到前一天的日期字符串。

`getStats()` 仅在 [StatsPage.tsx#L9](file:///workspace/src/pages/StatsPage.tsx#L9) 打开页面时调用一次，`byDay` 是 `Record<string, number>`，按对象插入顺序展示，无趋势、无补零、无"近 N 天"概念。

### 2.2 段落问题

[src/ai/prompts.ts#L12-L13](file:///workspace/src/ai/prompts.ts#L12-L13) `generatePrompt` 用 `profile.segments` 全量拼 `segDesc`：

```ts
const segDesc = profile.segments.map((s, i) => `第${i + 1}段「${s.title}」...`).join("\n");
```

[src/pages/GeneratePage.tsx](file:///workspace/src/pages/GeneratePage.tsx) 调用 `generateFeedback` 时无段落选择入参。规范档编辑页（[SpecProfilePage.tsx#L117-L138](file:///workspace/src/pages/SpecProfilePage.tsx#L117-L138)）可增删段落，但这是永久改动，无法满足"本次临时排除、下次默认全选"。

## 3. 架构

纯前端改动，无 schema 迁移，无新依赖。三个独立小改动：

- **时区**：抽 `localDay(ts)` helper（新文件 `src/lib/time.ts`），`useStats` 替换 `toISOString`。
- **趋势**：`Stats` 接口加 `recent7`，`StatsPage` 用纯 CSS 柱状条渲染近 7 天。
- **段落勾选**：`generatePrompt` / `generateFeedback` 加 `includedSegments` 入参；`GeneratePage` 加临时勾选 UI（`Set<number>`，不持久化）。

## 4. 时区修复设计

### 4.1 新文件 src/lib/time.ts

```ts
// 显式指定中国时区，不依赖运行环境系统时区
// 教培老师都在中国，固定 Asia/Shanghai 最严谨
const dayFmt = new Intl.DateTimeFormat("zh-CN", {
  timeZone: "Asia/Shanghai",
  year: "numeric", month: "2-digit", day: "2-digit",
});

/** 把毫秒时间戳转成中国时区的 YYYY-MM-DD 日期字符串 */
export function localDay(ts: number): string {
  const parts = dayFmt.formatToParts(new Date(ts));
  const y = parts.find(p => p.type === "year")!.value;
  const m = parts.find(p => p.type === "month")!.value;
  const d = parts.find(p => p.type === "day")!.value;
  return `${y}-${m}-${d}`;
}
```

### 4.2 useStats.ts 改动

[src/hooks/useStats.ts#L22](file:///workspace/src/hooks/useStats.ts#L22)：

```ts
// 旧：const day = new Date(u.timestamp).toISOString().slice(0, 10);
const day = localDay(u.timestamp);
```

顶部 `import { localDay } from "../lib/time";`。

## 5. 近 7 天每日趋势设计

### 5.1 Stats 接口扩展

[src/hooks/useStats.ts](file:///workspace/src/hooks/useStats.ts) `Stats` 增加：

```ts
export interface Stats {
  total: number;
  byType: Record<CallType, number>;
  byDay: Record<string, number>;        // 保留：全部历史，key 已修正为本地日
  recent7: { day: string; tokens: number }[];  // 新增：近 7 天升序，自动补零
}
```

### 5.2 recent7 计算逻辑（getStats 内）

```ts
const recent7: { day: string; tokens: number }[] = [];
const now = Date.now();
// 从 6 天前到今天，共 7 天，升序
for (let i = 6; i >= 0; i--) {
  const ts = now - i * 24 * 3600 * 1000;
  const day = localDay(ts);
  recent7.push({ day, tokens: byDay[day] ?? 0 });
}
```

说明：中国无夏令时，`now - i*86400000` 跨天安全；用 `localDay` 再转换确保日期归属正确，不依赖 UTC 边界。

### 5.3 StatsPage UI

[src/pages/StatsPage.tsx](file:///workspace/src/pages/StatsPage.tsx) "按日期"卡片改为：

- 顶部：近 7 天 CSS 柱状条（flex 横排 7 列，每列：固定高度容器 + 柱子 `height: tokens/max*100%`，柱子下方 `MM-DD` + 数值）
- `max = Math.max(...recent7.map(d => d.tokens), 1)`（至少 1 避免除零）
- 下方保留全部历史 `byDay` 列表（折叠或直接显示），供查看长期数据

无图表库依赖，纯 Tailwind + 内联 style 高度百分比。

## 6. 段落临时勾选设计

### 6.1 数据契约（types 不变）

`SpecSegment` 不加字段，临时选择不持久化。`SpecProfile.segments` 仍是段落模板定义。

### 6.2 prompts.ts 签名变更

[src/ai/prompts.ts#L12](file:///workspace/src/ai/prompts.ts#L12) `generatePrompt` 加第 5 个参数：

```ts
export function generatePrompt(
  profile: SpecProfile,
  student: Student,
  courseContent: string,
  history: Feedback[],
  includedSegments: SpecSegment[],   // 新增：本次实际参与生成的段落
) {
  // 用 includedSegments 替代 profile.segments 拼 segDesc
  const segDesc = includedSegments.map((s, i) =>
    `第${i + 1}段「${s.title}」约${s.targetWords}字，要点：${s.contentPoints}${s.freeNote ? "；补充：" + s.freeNote : ""}`
  ).join("\n");
  // 其余不变
}
```

锁字段逻辑（lockedFields）不受影响，仍读 `profile`。

### 6.3 generate.ts 透传

[src/ai/generate.ts](file:///workspace/src/ai/generate.ts) `generateFeedback` 入参加 `includedSegments`，透传给 `generatePrompt`：

```ts
export async function generateFeedback(args: {
  apiKey: string; profile: SpecProfile; student: Student;
  courseContent: string; history: Feedback[];
  includedSegments: SpecSegment[];   // 新增
}): Promise<{ feedback: string }> {
  const messages = generatePrompt(args.profile, args.student, args.courseContent, args.history, args.includedSegments);
  // 其余不变
}
```

### 6.4 GeneratePage UI 与状态

[src/pages/GeneratePage.tsx](file:///workspace/src/pages/GeneratePage.tsx)：

- 新增 state：`const [segIncluded, setSegIncluded] = useState<Set<number>>(new Set());`
- `useEffect([profileId])`：当 profileId 变化时，重置 `segIncluded` 为该 profile 所有段落索引全选：

```ts
useEffect(() => {
  const p = profiles.find(p => p.id === profileId);
  if (p) setSegIncluded(new Set(p.segments.map((_, i) => i)));
  else setSegIncluded(new Set());
}, [profileId, profiles]);
```

- UI：规范档下拉下方，当选中 profile 且段落 ≥1 时，显示"本次段落"区，每个段落一个 checkbox（label=`段落标题（约X字）`），勾选状态绑定 `segIncluded`。
- `doGenerate`：构造 `includedSegments`：

```ts
const includedSegments = profile.segments.filter((_, i) => segIncluded.has(i));
if (includedSegments.length === 0) { notify.error("请至少选择一个段落"); return; }
const out = await generateFeedback({ apiKey, profile, student, courseContent: text, history, includedSegments });
```

- "重新生成"按钮（preview 区，[GeneratePage.tsx#L134](file:///workspace/src/pages/GeneratePage.tsx#L134)）：**保留**当前 `segIncluded` 选择，不重置（用户调整勾选后重新生成应生效）。
- 生成按钮 disabled 条件不变（仍 `generating || !text || !studentId || !profileId`）；段落空集在 `doGenerate` 内 Toast 拦截。

### 6.5 BatchGeneratePage

[src/pages/BatchGeneratePage.tsx](file:///workspace/src/pages/BatchGeneratePage.tsx) 调用 `generateFeedback` 处，`includedSegments` 传该 profile 全部段落（批量场景不逐条勾选，保持原行为）：

```ts
const out = await generateFeedback({ apiKey, profile, student: ps.student, courseContent: ps.content, history, includedSegments: profile.segments });
```

## 9. 测试策略（TDD）

### 9.1 tests/lib/time.test.ts

```ts
describe("localDay", () => {
  it("北京时间 2026-06-27 00:00 归为 2026-06-27", () => {
    // UTC 2026-06-26 16:00 = 北京 2026-06-27 00:00
```

## 10. 边界与风险

| 场景 | 处理 |
|---|---|
| includedSegments 为空 | UI Toast 拦截"请至少选择一个段落"；generate 函数防御性允许（segDesc 为空，prompt 仍合法，但实际不会触达） |
| profile 切换 | segIncluded 重置为新 profile 全选 |
| 重新生成 | 保留当前 segIncluded（用户调整后重新生成应生效） |
| 批量生成 | 不逐条勾选，传 profile.segments 全量，保持原行为 |
| 近 7 天全为 0 | max=1，柱子高度 0%，显示空柱 + 数值 0，不报错 |
| 用户跨时区使用 | 固定 Asia/Shanghai，海外用户会看到中国日期；本项目目标用户在中国，可接受 |
| Intl 兼容性 | Intl.DateTimeFormat 所有现代浏览器支持，GitHub Pages 目标浏览器无问题 |

## 11. 验收标准

1. 凌晨 0–8 点生成的反馈，统计页 byDay 和 recent7 均归到当天（中国日期）。
2. 统计页显示近 7 天柱状趋势，无数据天显示 0 柱，日期升序。
3. 生成反馈页选中规范档后，下方显示段落勾选区，默认全选。
4. 取消勾选某段落 → 生成 → 反馈正文不含该段落对应结构（AI 实际是否提及受 prompt 控制，但 prompt 中无该段落引导）。
5. 切换规范档 → 勾选重置为新规范档全选。
6. 点"重新生成" → 保留当前勾选状态。
7. 取消全部段落 → 点生成 → Toast"请至少选择一个段落"，不发起请求。
8. 批量生成行为不变（全量段落）。
9. 全量测试通过，tsc clean，build 成功，线上 Playwright 验证通过。
